# LRU Cache 算法 —— 逐行讲解

## 概述

本文件实现了 **LRU（Least Recently Used，最近最少使用）缓存**。核心思想是：
- 当缓存容量满时，淘汰**最久未被访问**的数据。
- `get` 和 `put` 操作均要求 **O(1)** 时间复杂度。

数据结构组合了 **哈希表（字典）** 实现 O(1) 查找 + **双向链表** 维护访问顺序。

- **`key: int`**：类型注解，表示 `key` 参数应为 `int` 类型。用于缓存中唯一标识一条数据。

## 第 18–19 行：哨兵节点的创建

```python
self.head = ListNode(0, 0)
self.tail = ListNode(0, 0)
```

- `self.head` 和 `self.tail` 是**哨兵节点（sentinel node）**，也称为 **dummy 节点**。
- 它们的 `key` 和 `value` 无实际意义（填 0），只作为双向链表的**头尾边界标记**。
- **为什么用哨兵节点？**
  - 如果没有哨兵节点，当链表为空时，`head` 和 `tail` 都是 `None`，插入/删除的第一个节点需要做**空链表判断**，代码会充斥 `if self.head is None` 这样的分支。
  - 有了哨兵节点后，**链表永远非空**（至少有 head 和 tail 两个傀儡节点），插入和删除永远是"在中间操作"，不需要处理空链表特殊情况。

### 链表初始结构示意

```
head(哨兵) <--> tail(哨兵)
```

## 第 20 行：`self.head.next = self.tail`
- 将头哨兵的后继指针指向尾哨兵，形成 `head → tail` 的链。

---

## 第 21 行：`self.tail.prev = self.head`
- 将尾哨兵的前驱指针指向头哨兵，形成 `head ← tail` 的链。
- 至此链表初始化为：`head ⇄ tail`（双向连接）。

- 若不存在（缓存未命中），进入第 25 行。

- 将结果赋给局部变量 `node`。

- **`_` 前缀**：Python 约定，单下划线开头的方法/属性表示"私有"（约定而非强制），提示外部调用者不应直接使用。

- **`put`** 方法：写入/更新一条缓存数据。
- **`-> None`**：返回类型注解，该方法不返回有意义的值（返回 `None`）。

---

## 第 32 行：`if key in self.cache:`
- 判断 `key` 是否已存在于缓存中。（**已存在 / 不存在**是两个分支）

---

### 第 33–36 行：key 已存在的分支

```python
# 已存在：更新值 + 移到表头
node = self.cache[key]
node.value = value
self._move_to_head(node)
```

- **第 34 行**：从字典中取出已存在的节点。
- **第 35 行**：**原地更新**节点的 `value`——即使同一个 key，新值覆盖旧值。
- **第 36 行**：将该节点移到链表头部。因为此次 `put` 等同于一次"访问"，应标记为最近使用。

---

### 第 37–46 行：key 不存在的分支

```python
else:
    # 不存在：创建新节点
    new_node = ListNode(key, value)
    self.cache[key] = new_node
    self._add_to_head(new_node)

    # 超过容量，删除表尾节点（最久未使用）
    if len(self.cache) > self.capacity:
        tail_node = self._remove_tail()
        del self.cache[tail_node.key]
```

- **第 39 行**：调用 `ListNode(key, value)` 构造函数，创建一个全新的链表节点。
- **第 40 行**：将新节点插入字典 `self.cache` 中。**字典赋值即添加/覆盖**。
- **第 41 行**：调用 `_add_to_head` 将新节点添加到链表头部（最新位置）。

### 容量溢出检查

- **第 44 行**：`if len(self.cache) > self.capacity:`
  - **`len(self.cache)`**：字典长度 = 当前缓存中的条目数。
  - 若超出 `self.capacity`，则需淘汰最久未使用的节点。

- **第 45 行**：`tail_node = self._remove_tail()`
  - 调用 `_remove_tail()` 移除链表尾部的节点（即最久未使用的节点），并返回被移除的节点。

- **第 46 行**：`del self.cache[tail_node.key]`
  - **`del` 语句**：从字典中删除键为 `tail_node.key` 的条目。
  - **`tail_node.key`**：被淘汰节点的 key，用来索引字典中对应的键值对。
  - **为什么需要同时在链表和字典中删除？** 字典和链表是两套独立的引用，链表删除后节点对象还在，但字典中还保留着对该节点的引用。必须从字典中也删除，否则：
    1. `len(self.cache)` 不会减少，下次判断会继续超容。
    2. 后续 `get(key)` 可能拿到一个已从链表中移除的"僵尸节点"。

## 第 48–53 行：`_remove_node` 方法

```python
def _remove_node(self, node: ListNode):
    """从链表中移除节点"""
    prev_node = node.prev
    next_node = node.next
    prev_node.next = next_node
    next_node.prev = prev_node
```

### 参数
- **`node: ListNode`**：要移除的节点。

### 执行逻辑（图解）

移除前：
```
... <--> A <--> node <--> B <--> ...
```

1. **第 50 行**：`prev_node = node.prev` → 拿到 node 的前驱节点 A。
2. **第 51 行**：`next_node = node.next` → 拿到 node 的后继节点 B。
3. **第 52 行**：`prev_node.next = next_node` → 让 A 的 `next` 跳过 node，直接指向 B。
4. **第 53 行**：`next_node.prev = prev_node` → 让 B 的 `prev` 跳过 node，直接指向 A。

移除后：
```
... <--> A <--> B <--> ...
```

- **时间复杂度：O(1)**。仅修改 4 个指针，不涉及遍历。
- node 自身的 `prev`/`next` 仍然存在但已无关紧要（GC 会回收，因为没有任何引用了）。

## 第 55–60 行：`_add_to_head` 方法

```python
def _add_to_head(self, node: ListNode):
    """将节点添加到表头"""
    node.prev = self.head
    node.next = self.head.next
    self.head.next.prev = node
    self.head.next = node
```

### 作用
将一个节点插入到链表头部（哨兵 head 之后）。

### 执行逻辑（图解）

初始：
```
head <--> A <--> ...
```

目标：
```
head <--> node <--> A <--> ...
```

1. **第 57 行**：`node.prev = self.head` → node 的前驱设为 head。
2. **第 58 行**：`node.next = self.head.next` → node 的后继设为原来的 head.next（即 A）。
3. **第 59 行**：`self.head.next.prev = node` → A 的前驱设为 node（断开 A 与 head 的旧连接）。
4. **第 60 行**：`self.head.next = node` → head 的后继设为 node（断开 head 与 A 的旧连接）。

- **注意第 59 和 60 行的顺序**：必须先修改 `self.head.next.prev`（即 A.prev），再修改 `self.head.next`。因为第 60 行执行后，`self.head.next` 就变成 node 了，如果再通过它访问 `prev` 就会改错。
- **时间复杂度：O(1)**。

---

## 第 62–65 行：`_move_to_head` 方法

```python
def _move_to_head(self, node: ListNode):
    """将节点移到表头（标记为最近使用）"""
    self._remove_node(node)
    self._add_to_head(node)
```

### 执行逻辑
将一个**已存在于链表中的节点**移动到头部，分两步：
1. **`self._remove_node(node)`**：从当前位置摘下该节点。
2. **`self._add_to_head(node)`**：将该节点插入到头部。

- 这是 `_remove_node` + `_add_to_head` 的**组合调用**，实现"移动"效果。
- **时间复杂度：O(1)**（两步都是 O(1)）。

---

## 第 67–71 行：`_remove_tail` 方法

```python
def _remove_tail(self) -> ListNode:
    """移除表尾节点（最久未使用）"""
    tail_node = self.tail.prev
    self._remove_node(tail_node)
    return tail_node
```

### 返回类型
- **`-> ListNode`**：返回被移除的节点。

### 执行逻辑
1. **第 69 行**：`tail_node = self.tail.prev` → 拿到尾哨兵的前驱节点，即**最后一个真正存储数据的节点**（最久未使用）。
2. **第 70 行**：`self._remove_node(tail_node)` → 从链表中移除该节点。
3. **第 71 行**：`return tail_node` → 返回被移除的节点，供调用方（`put` 方法）获取其 `key` 以便从字典中删除。

- 由于有 `tail` 哨兵节点，表尾永远是 `self.tail.prev`，无需判空。

## 第 80–96 行：测试代码

```python
obj = LRUCache(2)
print("obj = LRUCache(2)")
```

- **第 80 行**：创建容量为 2 的 LRU 缓存实例。
  - 构造函数执行：`self.capacity = 2`，`self.cache = {}`，创建哨兵节点并建立 head ⇄ tail 双向连接。

### put(1, 1)
```python
obj.put(1, 1)
print("put(1, 1)")
```
- key=1 不在缓存中 → 创建 `ListNode(1, 1)` → 插入字典 → 加到链表头部。
- 链表：`head ⇄ (1,1) ⇄ tail`

### put(2, 2)
```python
obj.put(2, 2)
print("put(2, 2)")
```
- key=2 不在缓存中 → 创建 `ListNode(2, 2)` → 插入字典 → 加到链表头部。
- 链表：`head ⇄ (2,2) ⇄ (1,1) ⇄ tail`（2 是头部=最新，1 靠尾=较旧）

### get(1)
```python
print("get(1):", obj.get(1))  # 返回 1
```
- key=1 命中 → 取出 `(1,1)` 节点 → `_move_to_head` 将其移到头部。
- 链表变为：`head ⇄ (1,1) ⇄ (2,2) ⇄ tail`（1 变最新，2 变较旧）

### put(3, 3) —— 触发淘汰
```python
print("put(3, 3)")
obj.put(3, 3)  # 淘汰 key=2
```
- key=3 不在缓存中 → 创建 `ListNode(3, 3)` → 插入字典 → 加到头部。
- `len(self.cache)` = 3 > `self.capacity`(2) → **触发淘汰**。
- 调用 `_remove_tail()`：`self.tail.prev` 是 `(2,2)`（最久未使用）→ 从链表中删除，从字典中 `del self.cache[2]`。
- 链表：`head ⇄ (3,3) ⇄ (1,1) ⇄ tail`，字典：`{1: (1,1), 3: (3,3)}`

### get(2)
```python
print("get(2):", obj.get(2))  # 返回 -1（被淘汰）
```
- key=2 不在字典中 → `return -1`。

### put(4, 4) —— 再次触发淘汰
```python
print("put(4, 4)")
obj.put(4, 4)  # 淘汰 key=1
```
- key=4 不在缓存中 → 创建并插入 → 超容 → 淘汰 `tail.prev` 即 `(1,1)`。
- 链表：`head ⇄ (4,4) ⇄ (3,3) ⇄ tail`

### 后续 get 验证
```python
print("get(1):", obj.get(1))  # 返回 -1（被淘汰）
print("get(3):", obj.get(3))  # 返回 3
print("get(4):", obj.get(4))  # 返回 4
```

| 操作 | 结果 | 原因 |
|------|------|------|
| get(1) | -1 | 已被 put(4,4) 淘汰 |
| get(3) | 3 | 在缓存中 |
| get(4) | 4 | 在缓存中，且是最新 |

## 总结：复杂度分析

| 操作 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| get | **O(1)** | O(capacity) |
| put | **O(1)** | O(capacity) |

- `get`：字典查找 O(1) + 链表移动 O(1)
- `put`：字典查找/插入 O(1) + 链表插入/淘汰 O(1)

## 核心设计技巧

1. **哈希表 + 双向链表**：字典负责快速定位，链表负责维护顺序。
2. **哨兵节点**（head/tail dummy）：消除空链表边界判断，使所有插入/删除逻辑统一。
3. **每个节点同时存 key 和 value**：淘汰时需要通过 `tail_node.key` 从字典中删除对应条目。如果只存 value，就找不到字典中的键了。
4. **私有方法封装**（`_remove_node`、`_add_to_head` 等）：将底层链表操作抽离为小粒度方法，上层 `get`/`put` 只需调用组合即可，代码清晰易维护。
