# 快速选择（Quick Select）算法详解

---

## 一、什么是快速选择？

**快速选择（Quick Select）** 是一种在无序数组中查找**第 K 小（或第 K 大）元素**的算法，由 Tony Hoare 提出（他也提出了快速排序）。它是**快速排序（Quick Sort）的变体**——核心思路完全相同，但每次只递归处理包含目标的那一半，因此平均时间复杂度从 O(n log n) 降到了 **O(n)**。

### 一句话概括

> 像快速排序一样分区，但每次只往包含答案的那一侧递归，把"完整排序"变成了"定向查找"。

---

## 二、核心思想

### 2.1 从快速排序到快速选择

| 算法 | 每次分区后做什么 | 时间复杂度 |
|------|-----------------|-----------|
| **快速排序** | 对左右**两侧都**递归排序 | O(n log n) |
| **快速选择** | 只对包含目标的**一侧**递归 | **O(n)** 平均 |

### 2.2 分区（Partition）—— 算法的核心操作

分区操作用一个 **基准元素（pivot）** 将数组分为三部分：

```
[ 小于 pivot 的元素 ]   [ pivot ]   [ 大于 pivot 的元素 ]
         ↑                  ↑                ↑
      左侧区域           正确位置          右侧区域
```

分区完成后，pivot 落在了它的**最终排序位置** `i`，这意味着：
- pivot 就是数组中**第 i+1 小**的元素（0 索引）
- 左侧的 i 个元素 ≤ pivot
- 右侧的元素 > pivot

### 2.3 只往一侧递归

知道 pivot 是第 `i+1` 小之后，比较 `i+1` 与目标 `k`：

```
如果 i+1 == k  → 找到了！pivot 就是第 k 小的元素
如果 i+1 < k   → 目标在右边，只递归右侧 [i+1, r]
如果 i+1 > k   → 目标在左边，只递归左侧 [l, i-1]
```

这就是"快速选择"名字的由来——**快速**地**选择**第 K 个。

---

## 三、Lomuto 分区方案详解

最常用的分区方案有两种：**Lomuto** 和 **Hoare**。本实现使用 Lomuto 方案，它更直观易懂。

### 3.1 分区步骤图解

假设数组 `[4, 2, 7, 1, 5]`，选择最后一个元素 `5` 作为 pivot：

```
初始状态:
[4, 2, 7, 1, 5]
 ↑           ↑
 i=0       pivot

Step 1: j=0, arr[0]=4 ≤ 5 → 交换 arr[0]↔arr[0], i=1
[4, 2, 7, 1, 5]
    ↑        ↑
    i=1    pivot

Step 2: j=1, arr[1]=2 ≤ 5 → 交换 arr[1]↔arr[1], i=2
[4, 2, 7, 1, 5]
       ↑     ↑
       i=2 pivot

Step 3: j=2, arr[2]=7 > 5 → 不交换，i 保持 2
[4, 2, 7, 1, 5]
       ↑     ↑
       i=2 pivot

Step 4: j=3, arr[3]=1 ≤ 5 → 交换 arr[2]↔arr[3], i=3
[4, 2, 1, 7, 5]
          ↑  ↑
          i=3 pivot

Step 5: 循环结束，交换 arr[3]↔arr[4]（pivot 归位）
[4, 2, 1, 5, 7]
          ↑
       pivot 在索引 3
```

最终 pivot `5` 位于索引 3，左边 `[4,2,1]` 都 ≤ 5，右边 `[7]` > 5。

### 3.2 Lomuto 分区的代码模板

```python
def partition(arr, l, r):
    pivot = arr[r]          # 选最后一个元素为 pivot
    i = l                   # i 指向"≤pivot 区域"的下一个空位

    for j in range(l, r):   # j 扫描 [l, r-1]
        if arr[j] <= pivot:
            arr[i], arr[j] = arr[j], arr[i]
            i += 1          # 扩大 ≤pivot 区域

    arr[i], arr[r] = arr[r], arr[i]  # pivot 归位
    return i                # 返回 pivot 最终位置
```

### 3.3 为什么随机选 pivot？

如果数组**已经有序**且总是选最后一个元素作为 pivot：

```
[1, 2, 3, 4, 5]  选 5 → 分区后 pivot 在最右，只排除 1 个元素
[1, 2, 3, 4]     选 4 → 又只排除 1 个元素
...
```

每次只排除一个元素，退化为 O(n²)。

**随机化**选 pivot 使得期望时间复杂度回到 **O(n)**，最坏情况概率极低。

## 四、完整算法流程

### 4.1 算法步骤

```
quick_select(arr, l, r, k):
    1. 随机选 pivot，交换到末尾 r
    2. Lomuto 分区，pivot 落位到索引 i
    3. 判断：
       - 如果 i == k-1 → 返回 arr[:k]（前 k 个元素）
       - 如果 i < k-1  → 递归右侧 quick_select(arr, i+1, r, k)
       - 如果 i > k-1  → 递归左侧 quick_select(arr, l, i-1, k)
```

### 4.2 以本题为例：找离原点最近的 K 个点

```
points = [[3,3], [5,-1], [-2,4], [1,3], [-2,2]], k = 2

距离平方：dist = [18, 26, 20, 10, 8]

目标：找到距离最小的 2 个点
```

**第一轮分区**（随机 pivot，假设选中索引 4 即值 8）：

```
分区后（以 8 为基准）:
[3,3]  [-2,4]  [5,-1]  [1,3]  [-2,2]
 18      20      26      10      8

按距离分区 → 假设结果:
左侧(≤8): [[-2,2](8), [1,3](10)]
pivot 在索引 1
右侧(>8): [[3,3](18), [-2,4](20), [5,-1](26)]

pivot 是第 2 小（i+1 = 2），恰好等于 k=2
→ 返回前 2 个：[[-2,2], [1,3]] ✓
```

## 五、复杂度分析

### 5.1 时间复杂度

| 情况 | 复杂度 | 说明 |
|------|--------|------|
| **平均** | **O(n)** | 每次期望排除一半元素：n + n/2 + n/4 + ... = 2n |
| **最好** | O(n) | 第一次分区 pivot 恰好是第 k 个 |
| **最坏** | O(n²) | 每次只排除 1 个元素（有序数组 + 固定选端点），随机化后几乎不可能 |

**平均复杂度推导：**

```
T(n) = T(n/2) + O(n)   ← 每次递归处理一半
     = O(n) + O(n/2) + O(n/4) + ...
     = O(2n)
     = O(n)
```

### 5.2 空间复杂度

| 方面 | 复杂度 | 说明 |
|------|--------|------|
| **递归栈** | O(log n) | 每次递归区间减半，递归深度为 log n |
| **额外空间** | O(1) | 原地分区，不需要额外数组 |

### 5.3 与其他方法的对比

| 方法 | 时间复杂度 | 空间复杂度 | 适用场景 |
|------|-----------|-----------|---------|
| **完全排序** | O(n log n) | O(log n) | 需要全部有序 |
| **堆（Heap）** | O(n log k) | O(k) | k 很小 |
| **快速选择** | **O(n)** 平均 | O(log n) | **只需 Top K，不关心内部顺序** |

## 六、关键代码逐段解析

### 6.1 随机选 pivot 并交换到末尾

```python
pivot_idx = random.randint(l, r)          # 随机索引
points[pivot_idx], points[r] = points[r], points[pivot_idx]  # 换到末尾
pivot_dist = dist(points[r])              # 记录基准距离
```

> 把 pivot 藏到末尾，分区时扫描 [l, r-1]，最后再把 pivot 换回正确位置。

### 6.2 Lomuto 分区

```python
i = l                         # "≤pivot 区域"的右边界（开区间）
for j in range(l, r):         # j 从左到右扫描
    if dist(points[j]) <= pivot_dist:
        points[i], points[j] = points[j], points[i]  # 放到 ≤ 区域
        i += 1                # 扩大 ≤ 区域
points[i], points[r] = points[r], points[i]          # pivot 归位
```

### 6.3 递归方向判断

```python
if i + 1 == k:          # pivot 刚好是第 k 小
    return points[:k]   # 前 k 个就是答案（不一定有序）
elif i + 1 < k:         # 第 k 小在右边
    return quick_select(i + 1, r, k)
else:                   # 第 k 小在左边
    return quick_select(l, i - 1, k)
```

**核心理解**：`points[:k]` 返回的前 k 个元素**不一定有序**，但它们就是全局最小的 k 个。如果题目要求结果有序，需要额外排序（O(k log k)）。

## 七、快速选择的通用模板

```python
import random

def quick_select(arr, k):
    """
    在数组 arr 中找出第 k 小的元素（k 从 1 开始）
    返回第 k 小的元素值
    """
    def partition(l, r):
        # 随机选 pivot
        pivot_idx = random.randint(l, r)
        arr[pivot_idx], arr[r] = arr[r], arr[pivot_idx]
        pivot = arr[r]

        # Lomuto 分区
        i = l
        for j in range(l, r):
            if arr[j] <= pivot:
                arr[i], arr[j] = arr[j], arr[i]
                i += 1
        arr[i], arr[r] = arr[r], arr[i]
        return i

    def select(l, r, k):
        if l == r:
            return arr[l]

        pivot_pos = partition(l, r)

        if pivot_pos + 1 == k:
            return arr[pivot_pos]
        elif pivot_pos + 1 < k:
            return select(pivot_pos + 1, r, k)
        else:
            return select(l, pivot_pos - 1, k)

    return select(0, len(arr) - 1, k)


# 使用示例
arr = [3, 1, 4, 1, 5, 9, 2, 6]
print(quick_select(arr, 3))  # 第 3 小的元素 → 2
```

## 八、常见变体与应用

### 8.1 经典 Top K 问题

| 问题 | 解法 |
|------|------|
| 第 K 个最大元素 | 快速选择（调整比较方向） |
| 最小的 K 个数 | 快速选择 → `arr[:k]` |
| 最接近原点的 K 个点 | 快速选择（自定义距离比较函数） |
| 数组中第 K 个高频元素 | 快速选择（对频率做选择） |

### 8.2 第 K 大 vs 第 K 小

找**第 K 大**元素有两种思路：
1. **反转比较**：分区时 `arr[j] >= pivot`（大的放左边）
2. **转换索引**：第 K 大 = 第 `n - k + 1` 小

### 8.3 迭代实现（避免递归栈溢出）

```python
def quick_select_iterative(arr, k):
    l, r = 0, len(arr) - 1
    while l <= r:
        pivot_idx = random.randint(l, r)
        arr[pivot_idx], arr[r] = arr[r], arr[pivot_idx]
        pivot = arr[r]

        i = l
        for j in range(l, r):
            if arr[j] <= pivot:
                arr[i], arr[j] = arr[j], arr[i]
                i += 1
        arr[i], arr[r] = arr[r], arr[i]

        if i + 1 == k:
            return arr[i]
        elif i + 1 < k:
            l = i + 1
        else:
            r = i - 1
    return -1
```

迭代版本把递归替换为 `while` 循环，空间复杂度降为 **O(1)**。

## 九、常见疑问解答

### Q1: 快速选择和快速排序有什么区别？

快速选择是快速排序的"剪枝版"——快速排序每次分区后对**两侧**都递归，快速选择只递归**包含目标的那一侧**。这使时间复杂度从 O(n log n) 降到了 O(n)。

### Q2: 为什么不直接用排序？

如果只需要 Top K 且不关心内部顺序，排序做了多余的工作。排完整个数组需要 O(n log n)，而快速选择只需要 O(n)。

### Q3: 随机选 pivot 真的够吗？

对于绝大多数实际输入，随机化足够。如果需要严格的**最坏情况 O(n)**，可以使用 **BFPRT 算法**（中位数的中位数），但常数因子较大，实际应用中随机化快速选择更常用。

### Q4: 返回的 Top K 是有序的吗？

**不是。** 快速选择只保证前 k 个元素是全局最小的 k 个，但内部无序。如果题目要求有序结果，在快速选择后对 `arr[:k]` 排序即可：O(n + k log k)。

## 十、总结

| 维度 | 内容 |
|------|------|
| **算法本质** | 快速排序的变体，每次只递归一侧 |
| **核心操作** | Lomuto 分区 / Hoare 分区 |
| **关键优化** | 随机选 pivot，避免退化 |
| **平均时间** | O(n) |
| **最坏时间** | O(n²)（随机化后概率极低） |
| **空间** | O(log n) 递归栈（迭代版 O(1)） |
| **典型场景** | Top K 问题、第 K 大/小元素、中位数 |
| **延伸** | BFPRT 算法（严格 O(n) 最坏情况） |

> **记忆口诀**：快排分两边，快选走一边；随机选基准，平均 O(n) 稳。
