## 题目：最接近原点的 K 个点 (LeetCode 973)

给定平面上的一组点，找出离原点 `(0,0)` 最近的 `k` 个点。使用**快速选择算法**以 O(n) 平均时间复杂度求解。

---

## 第 1-2 行：导入模块

```python

import random
```

- **`import random`**：导入 `random` 模块，后续用于随机选取分区基准（pivot），避免快速选择算法在有序数据上退化到 O(n²)。


定义 `Solution` 类。这是 LeetCode 的标准答题模板，所有解法都封装在这个类中。

## 第 5 行：定义主方法

```python
def kClosest(self, points: List[List[int]], k: int) -> List[List[int]]:
```

- **`self`**：类实例的引用。
- **`points: List[List[int]]`**：参数 `points` 是一个二维列表，如 `[[1,3], [-2,2]]`，每个内层列表 `[x, y]` 代表一个二维坐标点。
- **`k: int`**：要找出的最近点的数量。
- **`-> List[List[int]]`**：返回值类型，返回离原点最近的 `k` 个点组成的列表。

---

## 第 6-7 行：距离计算函数

```python
def dist(p):
    return p[0]**2 + p[1]**2
```

这是定义在 `kClosest` 内部的**嵌套函数**。

- **`p`**：一个点坐标 `[x, y]`。
- **`p[0]**2 + p[1]**2`**：计算该点到原点的**欧几里得距离的平方**，即 `x² + y²`。

> **为什么不开根号？** 因为只需要比较距离的相对大小，开根号 `√(x²+y²)` 是单调递增变换，不影响大小关系。省去 `sqrt` 运算可以提升效率。

---

## 第 9 行：快速选择函数定义

```python
def quick_select(l, r, k):
```

定义递归的**快速选择**（Quick Select）函数，同样是嵌套函数。

- **`l`**：当前搜索区间的左边界索引（inclusive）。
- **`r`**：当前搜索区间的右边界索引（inclusive）。
- **`k`**：目标——找到前 `k` 个最小的元素。

> **快速选择**是快速排序的变体：每次只递归包含目标位置的那一半，平均时间复杂度 O(n)，最坏 O(n²)（但随机化 pivot 后概率极低）。

---

## 第 10-12 行：随机选择基准并交换到末尾

```python
# 随机选基准，避免最坏情况
pivot_idx = random.randint(l, r)
points[pivot_idx], points[r] = points[r], points[pivot_idx]
pivot_dist = dist(points[r])
```

- **`random.randint(l, r)`**：在 `[l, r]` 范围内随机选一个索引作为 pivot（基准元素）。
  > 如果总是选第一个或最后一个元素，当数据已有序时算法会退化为 O(n²)。随机化保证了**期望 O(n)**。
- **`points[pivot_idx], points[r] = points[r], points[pivot_idx]`**：将选中的 pivot 交换到区间末尾 `r` 位置。这是快速排序/快速选择的标准做法——先把 pivot 移到边缘，分区完成后再归位。
- **`pivot_dist = dist(points[r])`**：计算 pivot 点到原点的距离平方，分区时用它作为比较基准。

---

## 第 15-21 行：分区操作

```python
# 分区：左边<=pivot，右边>pivot
i = l
for j in range(l, r):
    if dist(points[j]) <= pivot_dist:
        points[i], points[j] = points[j], points[i]
        i += 1
points[i], points[r] = points[r], points[i]
```

这是 **Lomuto 分区方案**，将区间 `[l, r-1]` 的元素按与 pivot 的距离分为两部分：

1. **`i = l`**：`i` 是"小于等于 pivot 的区域"的右边界（下一个要放置的位置）。

2. **`for j in range(l, r)`**：`j` 从左到右扫描区间 `[l, r-1]`（注意不含 `r`，因为 `r` 存的是 pivot）。

3. **`if dist(points[j]) <= pivot_dist:`**：如果当前扫描到的点距离 ≤ pivot 的距离：
   - **`points[i], points[j] = points[j], points[i]`**：把它换到"≤ pivot 区域"的末尾（即位置 `i`）。
   - **`i += 1`**：扩大"≤ pivot 区域"的边界。

4. **循环结束后**：
   - 位置 `< i` 的所有元素：`dist ≤ pivot_dist`
   - 位置 `[i, r-1]` 的所有元素：`dist > pivot_dist`

5. **`points[i], points[r] = points[r], points[i]`**：将 pivot 从末尾 `r` 交换到正确位置 `i`。
   - 此时 pivot 在索引 `i`，左边全 ≤ 它，右边全 > 它。

---

## 第 23-29 行：递归判断

```python
# 递归分区
if i + 1 == k:
    return points[:k]
elif i + 1 < k:
    return quick_select(i + 1, r, k)
else:
    return quick_select(l, i - 1, k)
```

分区后，pivot 位于索引 `i`，说明它是数组中第 `i+1` 小的元素（因为索引从 0 开始）。

- **`if i + 1 == k:`**（`i == k - 1`）：
  pivot 恰好是第 `k` 小的元素。此时前 `k` 个元素（`points[:k]`）就是答案——**虽然它们内部不一定有序，但它们就是距离最小的 k 个点**。

- **`elif i + 1 < k:`**（`i < k - 1`）：
  第 `k` 小的元素在 pivot **右边**。只需要递归搜索右半部分 `[i+1, r]`，左半部分（包括 pivot）已确认在前 k 小之内。

- **`else:`**（`i > k - 1`）：
  第 `k` 小的元素在 pivot **左边**。递归搜索左半部分 `[l, i-1]`，右半部分可以丢弃。

---

## 第 31 行：启动快速选择

```python
return quick_select(0, len(points) - 1, k)
```

以整个数组的边界 `[0, n-1]` 启动快速选择递归，返回最终结果。

## 算法总结

| 方面 | 说明 |
|------|------|
| **算法** | 快速选择（Quick Select） |
| **核心思想** | 类似快速排序的分区，但每次只递归包含目标的一侧 |
| **时间复杂度** | 平均 O(n)，最坏 O(n²)（随机化 pivot 后几乎不可能） |
| **空间复杂度** | O(log n)（递归调用栈深度） |
| **适用场景** | Top K 问题，无需完全排序，只需找出前 K 个 |
