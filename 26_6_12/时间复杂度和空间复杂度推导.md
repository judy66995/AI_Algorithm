我来把时间复杂度和空间复杂度的推导过程，从定义到公式，一步一步讲透。

---

## 一、时间复杂度推导（QuickSelect）
### 1. 平均情况：O(n)
#### 核心假设
随机选 pivot 时，**每次分区都期望把数组分成两半**，每次递归只需要处理一半的元素。

#### 递推公式
设 `T(n)` 为处理长度为 `n` 的数组的时间复杂度：
```
T(n) = T(n/2) + O(n)
```
- `O(n)`：每次分区操作需要遍历数组，时间是线性的。
- `T(n/2)`：分区后，递归处理其中一半的数组。

#### 展开推导
我们把递推式逐层展开：
```
T(n) = O(n) + T(n/2)
     = O(n) + O(n/2) + T(n/4)
     = O(n) + O(n/2) + O(n/4) + T(n/8)
     = O(n) + O(n/2) + O(n/4) + O(n/8) + ... + O(1)
```
这是一个等比数列求和：
- 首项：`n`
- 公比：`1/2`
- 项数：`log₂n` 项

求和公式：
$$
n + \frac{n}{2} + \frac{n}{4} + \dots + 1 = 2n
$$
所以：
$$
T(n) = O(2n) = O(n)
$$

#### 直观理解
虽然分区操作每次都是 `O(n)`，但递归处理的数据量每次减半，总操作次数约等于 `2n`，依然是线性的。

---

### 2. 最好情况：O(n)
#### 场景
第一次分区时，pivot 恰好就是我们要找的第 `k` 小的元素。

#### 推导
- 只需要执行一次分区操作，时间复杂度就是 `O(n)`。
- 不需要任何递归，直接返回结果。

---

### 3. 最坏情况：O(n²)
#### 场景
每次分区时，pivot 都选到当前数组的最大/最小值，导致每次只能排除 1 个元素（比如有序数组+固定选端点）。

#### 递推公式
```
T(n) = T(n-1) + O(n)
```

#### 展开推导
```
T(n) = O(n) + T(n-1)
     = O(n) + O(n-1) + T(n-2)
     = O(n) + O(n-1) + O(n-2) + ... + O(1)
```
这是一个等差数列求和：
$$
n + (n-1) + (n-2) + \dots + 1 = \frac{n(n+1)}{2}
$$
所以：
$$
T(n) = O(n^2)
$$

#### 为什么随机选 pivot 后几乎不会出现？
随机选 pivot 时，每次都选到最大/最小值的概率极低，总概率是 `(1/n) * (1/(n-1)) * ...`，几乎可以忽略不计，因此工程上认为随机化 QuickSelect 的时间复杂度是 `O(n)`。

---

## 二、空间复杂度推导（QuickSelect）
### 1. 递归栈：O(log n)
#### 核心逻辑
每次递归，数组长度减半，递归深度就是 `log₂n`。

#### 推导
- 第一次递归：数组长度 `n`
- 第二次递归：数组长度 `n/2`
- 第三次递归：数组长度 `n/4`
- ...
- 直到数组长度为 1，递归结束

递归深度 `d` 满足：
$$
n / 2^d = 1 \implies d = \log_2 n
$$
所以递归栈的空间复杂度是 `O(log n)`。

> 如果用迭代实现（非递归），可以把递归栈优化掉，空间复杂度降为 `O(1)`。

---

### 2. 额外空间：O(1)
#### 说明
Lomuto 分区是**原地操作**：
- 不需要额外的数组来存储分区结果
- 只需要几个变量（`i`、`j`、`pivot`）来维护状态
- 所有交换都在原数组上进行

因此，额外空间复杂度是 `O(1)`。

---

## 三、总结对比
| 情况 | 时间复杂度 | 空间复杂度（递归） | 说明 |
| :--- | :--- | :--- | :--- |
| 平均 | `O(n)` | `O(log n)` | 随机 pivot，每次分半 |
| 最好 | `O(n)` | `O(1)` | 一次分区就命中目标 |
| 最坏 | `O(n²)` | `O(n)` | 每次只排除 1 个元素，递归深度为 `n` |
