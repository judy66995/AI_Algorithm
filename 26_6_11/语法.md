## 1. 类定义 `class`

```python
class TrieNode:
    ...
```

- **关键字** `class` 用于定义一个**类（class）**，类是创建对象的模板。
- `TrieNode` 是**类名**，按照 Python 的命名规范（PEP 8），类名使用 **大驼峰命名法**（每个单词首字母大写）。
- 类名后面的 `:` 表示类体的开始，缩进的部分都属于这个类。

> **类比理解**：如果说 `int`、`str`、`list` 是 Python 内置的“类型”，那么 `class` 就是让你自己定义新的类型。

## 3. `__slots__` 机制

```python
__slots__ = ("children", "is_end_of_word")
```

这是这段代码中**最不常见但最值得讲**的一个语法点。

### 普通对象的问题

Python 中，每个类的实例（对象）默认都有一个 `__dict__` 属性——这是一个字典，用于**动态存储**该实例的所有属性。

```python
class Normal:
    def __init__(self):
        self.a = 1
        self.b = 2

obj = Normal()
obj.c = 3  # ✅ 可以动态添加属性！
print(obj.__dict__)  # {'a': 1, 'b': 2, 'c': 3}
```

字典虽然灵活，但每个对象都要维护一个哈希表，**内存开销大**、**访问速度慢**。

### `__slots__` 做了什么

```python
class TrieNode:
    __slots__ = ("children", "is_end_of_word")
```

三件事：

1. **限制属性**：这个类的实例**只能**有 `children` 和 `is_end_of_word` 两个属性，不能再动态加别的。
2. **去掉 `__dict__`**：实例不再维护那个字典，属性直接存储在固定槽位里。
3. **节省内存 + 提高访问速度**：对于需要创建大量对象的场景（比如 Trie 树中有成千上万个节点），这个优化效果显著。

```python
n = TrieNode()
n.children = []
n.is_end_of_word = False
n.xxx = 1  # ❌ AttributeError: 'TrieNode' object has no attribute 'xxx'
```

> **注意**：`__slots__` 是以**类属性**的方式定义的（写在类里而不是 `__init__` 里），但它的作用是约束**实例**。

## 4. 实例方法 `def` 与 `self`

```python
def __init__(self):
    self.children = [None] * 26
    self.is_end_of_word = False
```

### `def` 关键字

`def` 用于定义**函数**。定义在类里面的函数，称为**方法（method）**。

### `self` 参数

- `self` 代表**调用这个方法的那个实例对象**。
- 当你写 `obj.method()` 时，Python 会自动把 `obj` 传给 `self`。
- `self` 不是关键字，只是一种**强烈的约定**——你可以用别的名字，但不要这样做。

### `__init__` —— 构造函数

- `__init__` 是一个**特殊方法**（magic method / dunder method），在对象被创建时**自动调用**。
- 它负责**初始化**新对象的属性，所以又叫**构造函数**。

```python
n = TrieNode()      # ① 创建一个空的 TrieNode 对象
                    # ② Python 自动调用 n.__init__()
                    # ③ n.children = ... ; n.is_end_of_word = False
                    # ④ n 现在是一个完整的、可用的对象
```

## 5. 列表初始化 `[None] * 26`

```python
self.children = [None] * 26
```

两个知识点：

### `[None]` —— 包含一个元素的列表

`None` 是 Python 中的**空值**，表示“什么都没有”。

### `* 26` —— 列表的重复运算符

列表 `*` 整数，会将列表的元素**重复**指定次数：

```python
[1] * 3        # → [1, 1, 1]
["a"] * 4      # → ["a", "a", "a", "a"]
[None] * 26    # → [None, None, None, ..., None]  # 共 26 个 None
```

在这个 Trie 实现中，每个节点有 26 个子节点槽位，分别对应 a～z 这 26 个小写字母。初始时全部为 `None`，表示还没有任何子节点。

- `True` 和 `False` 是 Python 的**布尔类型**（`bool`），只有这两个值。

## 7. 类型注解 `word: str` 与 `-> None`

```python
def insert(self, word: str) -> None:
def search(self, word: str) -> bool:
def startsWith(self, prefix: str) -> bool:
```

### 参数类型注解 `参数名: 类型`

```python
word: str      # 表示 word 参数应该是字符串类型
prefix: str    # 表示 prefix 参数应该是字符串类型
```

### 返回值类型注解 `-> 类型`

```python
-> None   # 表示这个函数不返回任何有意义的值（相当于 void）
-> bool   # 表示这个函数返回 True 或 False
-> str    # 表示这个函数返回字符串
-> int    # 表示这个函数返回整数
```

### 重要：类型注解不是强制的

Python 的**类型注解只是提示**，运行时不强制检查。下面的代码**不会报错**：

```python
obj.insert(123)  # 不会报错，但运行时 ord(123) 会报错
```

但在 IDE（如 VS Code）中，类型注解能提供**自动补全**和**静态检查**。

## 11. 字符串遍历 `for char in word`

```python
for char in word:
    index = ord(char) - ord('a')
```

- Python 的字符串是**可迭代对象**，可以直接放在 `for ... in` 中。
- 每次循环，`char` 会被赋值为字符串中的**下一个字符**。
- 例如 `"apple"` 会依次得到 `'a'`, `'p'`, `'p'`, `'l'`, `'e'`。

## 12. `ord()` 内置函数

```python
index = ord(char) - ord('a')
```

- `ord()` 是 Python 的内置函数，返回一个字符的 **Unicode 码点**（整数）。
- 对于英文字母：
  - `ord('a')` → `97`
  - `ord('b')` → `98`
  - ...
  - `ord('z')` → `122`

### 为什么用 `ord(char) - ord('a')`

这样可以将 26 个小写英文字母**映射到 0～25 的索引**，与 `children` 列表的 26 个槽位一一对应：

| 字符 | `ord(char)` | `ord(char) - ord('a')` |
|------|-------------|------------------------|
| `'a'`| 97          | 0                      |
| `'b'`| 98          | 1                      |
| ...  | ...         | ...                    |
| `'z'`| 122         | 25                     |

这就是**哈希映射**的简化版——用字母的位置作为数组下标，实现 O(1) 的查找。

## 13. 列表索引 `children[index]`


## 14. `not` 逻辑运算符

```python
if not node.children[index]:
```

- `not` 是 Python 的**逻辑非**运算符：把 `True` 变 `False`，`False` 变 `True`。
- 当用在非布尔值上时，Python 会先做**隐式布尔转换**：
  - `None` → `False`
  - `0`, `0.0` → `False`
  - 空字符串 `""` → `False`
  - 空列表 `[]`、空字典 `{}` → `False`
  - 其他值 → `True`

所以 `not node.children[index]` 等价于“这个槽位是 `None`（还没有子节点）”。

### 等价写法

```python
if not node.children[index]:
    # 等同于
if node.children[index] is None:
    # 等同于
if node.children[index] == None:
```

## 15. `if` 条件语句

```python
if not node.children[index]:
    node.children[index] = TrieNode()
```

- `if 条件:` —— 如果条件为 `True`，就执行下面缩进的代码块。
- Python 用**缩进**（通常为 4 个空格）来划分代码块，而不是用 `{}`（与 C/Java/JavaScript 不同）。
- 这里的逻辑：如果当前字符对应的子节点不存在，就先创建一个新节点。

## 16. `return` 返回语句

```python
return False
return True
return node.is_end_of_word
```

- `return` 用于**从函数中返回一个值**，并**立即结束函数的执行**。
- `return` 后面的表达式会被计算，然后这个值被返回给调用者。
- 如果函数执行到末尾都没有遇到 `return`，则隐式返回 `None`。

```python
def search(self, word: str) -> bool:
    node = self.root
    for char in word:
        index = ord(char) - ord('a')
        if not node.children[index]:
            return False      # 中途发现字符匹配不上，直接返回 False
        node = node.children[index]
    return node.is_end_of_word  # 遍历完所有字符，返回是否为完整单词
```

## 17. `if __name__ == "__main__":` 入口守卫

```python
if __name__ == "__main__":
    # 测试代码...
```

这是 **Python 中最经典、最重要的惯用法之一**。

### 工作原理

- 每个 Python 文件（模块）都有一个内置变量 `__name__`。
- 当你**直接运行**这个文件时（例如 `python main.py`），`__name__` 的值是字符串 `"__main__"`。
- 当这个文件被**其他文件导入**时（例如 `import main`），`__name__` 的值是这个文件的模块名 `"main"`。

### 为什么要这样写

```python
# main.py
class Trie:
    ...

if __name__ == "__main__":
    obj = Trie()
    obj.insert("apple")
    print(obj.search("apple"))
```

- **直接运行** `python main.py`：`__name__ == "__main__"` 成立 → 执行测试代码 ✅
- **被导入** `import main`：`__name__ == "main"` → 不执行测试代码，只导入 `Trie` 类 ✅

这样，同一个文件**既可以作为独立的程序运行，也可以作为模块被其他代码引用**。

## 18. `print()` 输出函数

```python
print("obj = Trie()")
print("search(\"apple\"):", obj.search("apple"))
```

- `print()` 是 Python 的内置函数，将内容输出到**标准输出**（通常是终端/控制台）。
- 可以接收多个参数，用逗号分隔，输出时自动用空格隔开：

```python
print("结果:", 42)  # 输出: 结果: 42
```

- `\"` 是**转义字符**，用来在双引号字符串中表示一个字面的双引号：

```python
print("He said \"Hello\"")  # 输出: He said "Hello"
```

## 附：完整执行流程演示

假设调用 `search("app")`：

```
Trie 中已有: apple

search("app") 被调用
│
├─ node = root（根节点）
│
├─ char = 'a' → index = 0
│   └─ children[0] 存在 → node = 'a' 节点
│
├─ char = 'p' → index = 15
│   └─ children[15] 存在 → node = 'p' 节点
│
├─ char = 'p' → index = 15
│   └─ children[15] 存在 → node = 第二个 'p' 节点
│
└─ 循环结束，检查 node.is_end_of_word
    └─ 此时 node 是 "apple" 的 'p' 节点，is_end_of_word = False
       （因为只有 "apple" 的 'e' 才是单词结尾）
    └─ 返回 False ❌  （"app" 不是完整单词，只是前缀）
```
